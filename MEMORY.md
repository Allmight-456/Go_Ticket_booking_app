## What This Project Is

A Go REST API backend for concurrent event-ticket booking. Originally a CLI learning exercise,
revamped into a production-architecture backend demonstrating:
- JWT-based RBAC (admin / user roles)
- Conflict-safe booking (Redis lock + PostgreSQL `SELECT FOR UPDATE` + optimistic locking)
- Immutable audit trail with field-level diff
- Redis sliding-window rate limiting (Lua scripts)
- Batch operations via pgx Batch API

**Module**: `github.com/Allmight-456/ticketflow`
**Entry point**: `cmd/api/main.go`

## Architecture (Clean Architecture, inward dependencies only)

```
Handler → Service → Repository → Domain
                              ↑
                    (domain structs flow through all layers)
```

| Layer | Path | Rule |
|-------|------|------|
| Domain | `internal/domain/` | Pure structs + sentinel errors. Zero external deps (except uuid, jwt). |
| Repository | `internal/repository/postgres/` `internal/repository/cache/` | Data access only. No business logic. |
| Service | `internal/service/` | Business logic. Calls repos. Never touches HTTP. |
| Handler | `internal/handler/` | HTTP only. Parses request, calls service, renders JSON. |
| Middleware | `internal/middleware/` | Cross-cutting: JWT parse, RBAC, rate limit. |
| Router | `internal/router/router.go` | Wires all routes and middleware chains. |

## Design Decisions Worth Remembering

1. **Dual-lock strategy**: Redis lock (prevents thundering herd) + `FOR UPDATE` (authoritative DB serialisation). Both are needed.
2. **Optimistic lock** (`version` column on events): `UPDATE … WHERE version = $old` fails if another TX already incremented it — third layer of safety.
3. **Audit log in same TX as booking**: `LogInTx()` ensures the audit record and the booking commit or rollback together. No orphaned audit entries.
4. **Partial unique index** on `bookings(user_id, event_id) WHERE status IN ('pending','confirmed')`: prevents double-booking while allowing re-booking after cancellation.
5. **Soft delete** on events: `deleted_at` column + partial indexes. Historical bookings still reference the event row.
6. **Money as integer cents**: Never `FLOAT` for money. `price_cents INT`, `total_price_cents BIGINT`.
7. **`internal/` directory**: Go compiler-enforced visibility — nothing outside this module can import these packages.
8. **Rate limiter fails open**: If Redis is unavailable, rate limiting is skipped rather than blocking all traffic.
